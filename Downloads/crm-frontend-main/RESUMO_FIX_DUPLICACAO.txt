┌──────────────────────────────────────────────────────────────────────────────┐
│ 🔴 PROBLEMA IDENTIFICADO E CORRIGIDO                                         │
└──────────────────────────────────────────────────────────────────────────────┘

SINTOMA:
  ❌ Lead aparece uma vez ao carregar
  ❌ Depois de alguns segundos DUPLICA
  ❌ Pode ter múltiplas cópias do mesmo lead
  ❌ Happens both when creating in CRM and via Supabase

CAUSA RAIZ:
  ❌ addLead() SEMPRE gerava novo UUID mesmo quando lead já tinha ID
  ❌ Isso fazia o mesmo lead ser criado com dois IDs diferentes
  ❌ Fetch inicial + Realtime INSERT processavam o mesmo lead 2 vezes

CENÁRIO DE DUPLICAÇÃO:
  1. Criar lead: id = "abc123"
  2. addLead() ignora ID e gera novo: id = "xyz789" ← BUG!
  3. Salva 2 registros diferentes em Supabase
  4. Realtime vê INSERT e adiciona novamente
  5. Resultado: 2 leads visualmente iguais, mas com IDs diferentes

┌──────────────────────────────────────────────────────────────────────────────┐
│ ✅ SOLUÇÃO IMPLEMENTADA                                                      │
└──────────────────────────────────────────────────────────────────────────────┘

1. useCrmData.ts - addLead() [CRÍTICA]
   ✅ Respeita ID existente: id: lead.id || crypto.randomUUID()
   ✅ Verificação dupla: if (lead.id && leads.some(l => l.id === lead.id))
   ✅ Usa criadoEm existente: criadoEm: lead.criadoEm || new Date()
   ✅ Resultado: Impossível criar lead duplicado com mesmo ID

2. useSupabaseLeads.ts - INSERT Handler
   ✅ Dupla verificação: const existingLead = crm.leads.find(l => l.id === lead.id)
   ✅ Logging: console quando detecta duplicação
   ✅ Remove fallbacks: nome vazio, pipeline/stage vazios (não "Lead importado")
   ✅ Resultado: INSERT realtime não duplica

3. useSupabaseLeads.ts - UPDATE Handler
   ✅ Verifica mudanças: só atualiza campos que realmente mudaram
   ✅ Mais eficiente: evita re-renders desnecessários
   ✅ Resultado: Sem atualizações fake

4. useSupabaseLeads.ts - deduplicateLeads()
   ✅ Nova função que remove leads duplicados existentes
   ✅ Executada após loadLeads() inicial
   ✅ Limpeza automática de duplicatas antigas
   ✅ Resultado: Se houver duplicatas antigas, são removidas
